Guard: Ubuntu 懒人初始化 & 地域防御脚本
这玩意儿是我自己用的。每次买台新 VPS，最烦的就是重复敲那几行 apt update、装工具，还得防着全世界的扫描器来爆破我的 SSH。

所以我干脆把这些活儿全揉进了一个脚本里。它不光能帮你把常用的顺手工具装好，最重要的是能利用 ipset 在底层筑起一道“地域墙”——除了中国大陆的 IP，谁也别想碰你的 22 端口。

它能帮你干啥？
补齐“瑞士军刀”：自动装好 curl、git、vim、htop、lsof 这些排错必备的小工具。

物理隔绝海外流量：直接在系统内核层面把 8000 多个国内 IP 段做成白名单，海外那些乱七八糟的扫描机器人在第一关就被踹飞了。

不嫌弃你的环境：不管你习惯用 UFW 还是喜欢原生 iptables，脚本都能自动识别并把规则“焊”进去。

自己会“进化”：脚本会在系统里留个小闹钟，每个月 1 号凌晨自动更新国内 IP 库，不用你操心。

怎么用？
在新买的 Ubuntu 机器上，直接复制下面这行代码丢进去敲回车就行（记得用 root 权限）：

Bash
bash <(curl -sL https://raw.githubusercontent.com/Yunzexinghai/guard/main/init.sh)
常见疑问（Q&A）
Q：我会不会把自己锁在外面？
A： 不会。脚本写得很稳，优先放行了你当前的连接。只要你是从国内连过去的，百分之百没问题。

Q：它会弄乱我的网站或者 Docker 吗？
A： 放心，它只盯着 22 端口（SSH）看。你跑的网页（80/443）或者 Docker 容器的路由，该怎么走还怎么走，互不干扰。

Q：我想改里面的小工具怎么办？
A： 既然都看这个仓库了，直接改 init.sh 顶部的 COMMON_TOOLS 变量就行，想加啥删啥随你便。
